## Памятка по работе с Git
---
### Работа с файловой системой:
1. ~~~pwd~~~ — показать рабочую папку (print working directory)
2. cd ~ — сменить директорию на домашнюю
3. ls - отобразить содержимое директории
4. ls -a - показать все файлы, в том числе скрытые
5. touch %ИМЯ_ФАЙЛА% - создать файл
6. mkdir new-dir - создать директорию
7. cp [что_копируем] [куда_копируем] - копирование файлов и директорий (источников "что копируем" может быть несколько). Копирование каталогов необходимо выполнять с флагом -r
8. mv - перемещение файла или папки
9. cat - чтение и вывод текстового файла
10. rm - удаление файла
11. rmdir - удаление пустого каталога
12. rm -r - удаление каталога вместе с вложенными файлами
---
### Инициализация Git:
1. git init - создать репозиторий Git
2. rm -rf .git - отключить отслеживание репозитория Git
3. git status - показать состояние репозитория
4. git add --all - подготовить к сохранению все файлы в каталоге
5. git add [имя файла] - подготовить к сохранению определенный файл
6. git commit -m "заметка" - выполнить коммит, для которого установить краткую заметку (описание изменений)
7. git log - посмотреть историю коммитов
---
### Работа с удаленным репозиторием GitHub:
1. ls -la .ssh/ - вывести список созданных ssh ключей (как правило, в домашнем каталоге)
2. sh-keygen -t ed25519 -C "электронная почта, к которой привязан ваш аккаунт на GitHub" - генерация SSH ключа ed25519, обычно требуется 1 раз
3. ssh-keygen -t rsa -b 4096 -C "электронная почта, к которой привязан ваш аккаунт на GitHub" - вместо алгоритма ed25519 можно использовать RSA
4. clip < ~/.ssh/id_ed25519.pub - скопировать публичный ключ ed25519 в буфер обмена для последующей вставки в GitHub
5. ssh -T git@github.com - проверка корректности настройки SSH
6. git remote add origin "адрес репозитория GitHub" - привязать удаленный репозиторий к локальному (выполняется в каталоге проекта)
7. git remote -v - проверить, что локальный и удаленный репозиторий связаны
8. git push -u origin master (или main) - отправить изменения на удаленный репозиторий, в последующие разы достаточно использшовать только "git push"
9. git clone - клонировать репозиторий с GitHub в локальный Git
